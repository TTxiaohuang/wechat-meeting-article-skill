# Input Contract

Create `article.json` with UTF-8 encoding. Required top-level keys are `_meta`, `meta`, and `sections`; recommended top-level keys are `theme`, `template`, `palette`, `custom_sections`, and optionally `assets`.

**Writing `article.json` directly with the file-writing tool is the recommended default.** After writing, immediately run the renderer to validate the JSON and catch syntax errors. To avoid JSON quote errors with Chinese text:

- Replace Chinese quotation marks `""` inside JSON string values with escaped Unicode `“` and `”`, or avoid them by rephrasing.
- Use `《》` for book/journal titles instead of `""` when both appear in the same string.
- Always validate by running the renderer immediately after writing.

If the environment reliably handles UTF-8 Python source files, an alternative is to create `article_data.py` with an `ARTICLE` dict and dump it:

```bash
python scripts/write_article_json.py article_data.py --out article.json
```

Use `scripts/update_article_gate.py` to patch required intake metadata into an existing valid `article.json`:

```bash
python scripts/update_article_gate.py article.json --material-folder path/to/material-folder --date 2026-05-28 --editor "推文编辑" --template classic --palette classic
```

When extracted materials already exist, you may create a scaffold for inventory support:

```bash
python scripts/draft_article_from_materials.py extracted_materials --out article.scaffold.json
```

Do not deliver `article.scaffold.json` directly. It is generated by deterministic heuristics and can omit large amounts of content. The final `article.json` must be expanded by the agent after reading the extracted materials.

## Rules

- Keep `text`, `summary`, `intro`, and `closing` as plain text. The renderer escapes HTML.
- Use arrays even when there is only one paper, comment, speech, or viewpoint.
- Keep `_meta.intake_gate` in the final article. Required decisions are `material_folder`, `date`, `editor`, and `visual_style`.
- If editor credit should be omitted, set `_meta.intake_gate.editor.status` to `omitted_confirmed`; otherwise set `meta.editor`.
- If the default visual style is accepted, set `_meta.intake_gate.visual_style.status` to `default_confirmed`.
- Put incomplete or uncertain fields as empty strings. Ask for confirmation when the missing field affects publication accuracy or privacy.
- If source materials conflict, preserve the conflict in notes and ask for confirmation instead of silently merging.
- Keep source paths relative to the input material folder when possible.
- Treat `source` as private trace metadata for verification only; the renderer does not show it in the WeChat body. Create `source_trace.md` only when the user asks for a traceability or audit package.
- For English speeches, `mode: "full_text"` means the card should preserve the original speech draft. Use this as the default when speech drafts are provided.
- For literature sharing, prefer `background`, `research_question`, `methods_data`, `findings`, and `discussion_value` over a short generic `summary`.
- `images` can contain strings or objects like `{"url": "path-or-url", "caption": "图注", "alt": "替代文本"}`. Use only supplied or explicitly generated images.
- Put supplied images in `cover_image`, section-level `images`, or paper-level `images`. For WeChat import, prefer image URLs already uploaded to a reliable host/editor, or local paths only for preview before manual upload.
- For English speech cards, use the optional `photo` field to specify a portrait image for each speaker (e.g., `"photo": "edward.jpg"`). The renderer displays it as a small circular avatar above the speaker name. Use relative paths from the material folder.
- Use `--embed-images` flag when rendering to embed local images as base64 data-uri in the HTML. This ensures images display in the browser preview and are automatically uploaded to WeChat CDN when pasted. Without this flag, local image paths must be resolvable from the HTML file's location.
- **Item-level images**: comments, discussion items, and viewpoints support an optional `images` array. Images are rendered at the bottom of the quote block, after the text. Example: `{"speaker": "同学", "text": "...", "images": ["result.png"]}`.
- **Inline image markers**: text fields can contain `{{image:src}}` or `{{image:src|caption}}` markers to insert images at specific positions within the text. The renderer splits the text at markers and inserts the image between paragraphs. Example: `"text": "结果如下：\n\n{{image:table1.png|回归结果}}\n\n可以看出..."`. Inline markers work in any text field rendered by `paragraphs()`, including paper descriptions, summaries, comments, and discussion items.
- Omit sections with no source material instead of filling placeholder text.
- Use `custom_sections` only for occasional inserts inside the reading-sharing meeting flow. Supported insert `type` values are `honor_news`, `announcement`, `milestone`, and `note`.
- Supported `placement` values are `after_lead`, `before_english_exchange`, `after_english_exchange`, `before_literature_sharing`, `after_literature_sharing`, `before_policy_discussion`, `after_policy_discussion`, `before_free_discussion`, `after_free_discussion`, and `before_closing`. If omitted, the renderer uses `after_lead`.
- For `honor_news`, include the clearest available `person`, `award`, `organization`, `date`, and `text`. Ask before publishing if the honor news contains private photos, certificate numbers, uncertain names, or incomplete award details.
- Set `"theme": "zhengeryanzi"` to enable the default static brand visual treatment: small inline SVG section marks and closing signature. It does not add a top brand card by default.
- Set `"template"` to one reading-sharing visual style: `"classic"`, `"notebook"`, `"journal"`, `"campus"`, `"minimal"`, `"magazine"`, `"warm-note"`, `"briefing"`, or `"fieldnote"`.
- Set `"palette"` to `"classic"`, `"forest"`, `"blueprint"`, `"warm"`, `"ink"`, `"sunrise"`, or `"mono"`. Use only one palette per article unless you are manually revising the renderer.
- Before delivery, run `scripts/check_article_json.py article.json --html dist/article.wechat.html` and fix publication-relevant warnings. Treat every reported issue as blocking unless the user explicitly accepts that exact risk.
- If `_meta.scaffold_generated` is present, the file is not a final article and must be rewritten or expanded into a separate `article.json`.
- JSON requires standard double quotes around keys and string values. Do not paste unescaped quotes inside strings. For large articles, prefer `write_article_json.py` or Python `json.dumps(..., ensure_ascii=False, indent=2)` when the environment preserves UTF-8. In Claude Code on Windows, avoid giant Bash heredocs and inline Python; direct file writing plus immediate render/check is often more reliable.

## Manual WeChat Fields

Copying rendered HTML into the WeChat editor only fills the body. In the final response, provide short manual fields instead of creating a required metadata file:

- 公众号标题: use `meta.title` or a polished variant.
- 摘要: use a one-sentence digest based on `meta.summary`.
- 封面建议: use a supplied meeting photo, PPT title slide, award certificate, or "暂无合适封面素材".
