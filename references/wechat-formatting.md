# 微信排版规范

公众号文章以移动端为主，微信编辑器可能过滤 HTML 和 CSS。优先使用保守的内联样式，避免 JavaScript。

## 推荐的交付方式

1. 主要方式：将 `article.wechat.html` 复制到微信编辑器、doocs/md、135 编辑器、秀米等。
2. 更安全的自动化：通过微信 API 创建草稿，预览后手动发布。
3. 除非用户明确要求并接受操作风险，否则不要自动发布。

## 导入微信

不要将 `article.wechat.html` 的源代码直接粘贴到微信编辑器，否则会显示可见的 HTML 代码。

推荐的手动导入方式：

1. 在浏览器中打开 `article.preview.html`
2. 选中渲染后的文章内容
3. 复制
4. 粘贴到公众号编辑器
5. 手机预览确认后发布

替代方式：

- 将 `article.wechat.html` 导入 doocs/md、135 编辑器、秀米等支持 HTML 源码导入的编辑器。
- 导入后检查横向滚动卡片和评论样式，编辑器可能会过滤部分样式。

## 布局规则

- 仅使用内联样式。
- 使用简单的块级元素：`section`、`p`、`span`、`strong`、`em`、`blockquote`、`img`。
- 避免外部 CSS、JavaScript、iframe、复杂定位和远程字体依赖。
- 段落简短，`line-height` 设为 `1.75` 左右。
- 不要依赖桌面宽度，按 360-430px 宽度测试。
- 不要在正文中显示提取元数据、源文件名或本地文件路径。
- 用导读摘要卡、编号章节标题、论文信息块和引用块来避免类似报告的原始排版。

## 英语发言卡片

默认使用横向滚动 HTML 卡片：

- 容器：`overflow-x:auto; white-space:nowrap;`
- 卡片：`display:inline-block; vertical-align:top; width:92%; max-width:340px; white-space:normal;`
- 每人一张卡片。
- 有发言稿时保留英文原文全文。
- 卡片前加"左右滑动查看完整英文发言"提示。
- 发言人头像 36px 圆形，小巧不抢眼。

## 文献分享

每篇论文使用两段式布局：

1. **摘要卡**（暖色底，带边框）：论文标题、作者、期刊、主讲人，以及核心发现的要点列表。
2. **详情字段**（左边框块）：研究背景、研究问题、方法与数据、讨论价值——每个字段在紧凑的边框区域内。
3. **讨论卡片**（横向滚动）：每条评论一张卡片，显示发言人姓名和内容。与英语交流卡片样式一致。

## 自由讨论

自由讨论环节使用：

1. **简短 intro**（1-2 句话）：概括讨论的话题，让读者快速了解。
2. **横向滚动卡片**：每人一张卡片，样式同英语交流和文献讨论卡片。
3. **会议小结**（边框卡片）：有 closing 时渲染。

自由讨论的内容不能与文献讨论评论重复。编辑提炼规则详见 `editorial-style.md`。

## SVG 说明

SVG 用于渲染器内置的装饰元素：章节标记、引用标记、发言人头像。这些是自包含的内联 SVG，可安全通过微信编辑器导入。用户无需操作。

自定义 SVG 交互或动画需保持自包含，避免脚本、外部资源、滤镜和脆弱的 ID。提供非 SVG 的 HTML 卡片回退。仅在用户明确要求时使用。

## 动效说明

动效是可选的实验性功能。仅在 `experimental_motion` 为 true 且用户同意预览时使用。优先使用微小的自包含 SVG 或 CSS 动效，降级为静态分割线、标记或章节标记。不要使用 JavaScript、滚动触发效果、外部资源或隐藏文本的动效。

## 视觉风格

使用克制的学术风格：

- 正文：近黑色，非纯黑。
- 主题色：一个柔和的蓝或绿，搭配浅色中性背景。
- 章节标题：清晰但紧凑。
- 引用/评论：左边框或浅色背景，不要用过大的装饰卡片。
- 默认的公众号署名风格保持内联 SVG 装饰静态克制：章节标记、引用标记和结尾署名。除非明确要求并预览测试，否则不要加动画。
- 支持的模板：`classic`、`notebook`、`campus`、`magazine`、`briefing`；旧名称通过别名映射。支持的配色：`classic`、`forest`、`blueprint`、`warm`、`ink`、`sunrise`、`mono`、`sakura`、`ocean`。这些是内联样式变体，设计为可安全通过微信编辑器导入。

## 可选插页块

插页视觉上要突出但简短：

- 喜讯：紧凑的祝贺块，带小标记、奖项/人物元数据、可选的证书或照片、简短祝贺段落。
- 通知：通知块，带标题、日期或目标受众、简短的行动说明。
- 里程碑：低调的进展块，带里程碑标题、日期、简短说明。
- 备注：用于上下文补充的小型附注块。
