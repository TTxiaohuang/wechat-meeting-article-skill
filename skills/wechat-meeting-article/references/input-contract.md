# Input Contract

Create `article.json` with UTF-8 encoding. Required top-level keys are `meta`, `sections`, and optionally `assets`.

Prefer creating the starter file with:

```bash
python scripts/create_article_json.py --out article.json
```

When extracted materials already exist, you may create a scaffold for inventory support:

```bash
python scripts/draft_article_from_materials.py extracted_materials --out article.scaffold.json
```

Do not deliver `article.scaffold.json` directly. It is generated by deterministic heuristics and can omit large amounts of content. The final `article.json` must be expanded by the agent after reading the extracted materials.

## Rules

- Keep `text`, `summary`, `intro`, and `closing` as plain text. The renderer escapes HTML.
- Use arrays even when there is only one paper, comment, speech, or viewpoint.
- Put incomplete or uncertain fields as empty strings and add a note in `source_trace.md`.
- If source materials conflict, preserve the conflict in notes and ask for confirmation instead of silently merging.
- Keep source paths relative to the input material folder when possible.
- Treat `source` as private trace metadata. It is for `source_trace.md` and verification only; the renderer does not show it in the WeChat body.
- For English speeches, `mode: "full_text"` means the card should preserve the original speech draft. Use this as the default when speech drafts are provided.
- For literature sharing, prefer `background`, `research_question`, `methods_data`, `findings`, and `discussion_value` over a short generic `summary`.
- `images` can contain strings or objects like `{"url": "path-or-url", "caption": "图注", "alt": "替代文本"}`. Use only supplied or explicitly generated images.
- Put supplied images in `cover_image`, section-level `images`, or paper-level `images`. For WeChat import, prefer image URLs already uploaded to a reliable host/editor, or local paths only for preview before manual upload.
- Omit sections with no source material instead of filling placeholder text.
- Set `"theme": "zhengeryanzi"` to enable the default static brand visual treatment: masthead, small inline SVG section marks, and closing signature.
- Before delivery, run `scripts/check_article_json.py article.json --html dist/article.wechat.html` and fix publication-relevant warnings.
- If `_meta.scaffold_generated` is present, the file is not a final article and must be rewritten or expanded into a separate `article.json`.
- JSON requires standard double quotes around keys and string values. Do not paste unescaped quotes inside strings. For example, write `围绕'最近最想做的事情'展开分享` or escape quotes as `围绕\"最近最想做的事情\"展开分享`.
- Do not handwrite large JSON blocks when a script can generate them. Use `json.dumps(..., ensure_ascii=False, indent=2)` in Python when building `article.json` programmatically.
